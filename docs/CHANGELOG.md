## [0.9.0] — 2026-06-12 ("DeveloperOS in its own window")
> Why this release exists: slice 16 changed how launching DeveloperOS *feels* — a standalone
> desktop window instead of a browser tab — and the downloadable assets must match what the
> code does. The Setup/exe attached to this release open the window experience.

### Added
- **Project version bumped to 0.9.0**; shown in `devos --version` and the dashboard's System status.
- **DeveloperOS now opens in its own window (slice 16).** Launching DeveloperOS (`devos app` or `DeveloperOS.exe`) no longer drops you into a browser tab with an address bar — it opens a **standalone desktop window** with its own taskbar entry, like a normal app. Under the hood this uses the app mode of Edge (which every Windows 10/11 machine already has; Chrome as fallback), so it adds **zero** new runtime, dependencies, or network surface — the window is simply your browser showing the same local, loopback-only dashboard without any chrome. If no app-capable browser is found it falls back to your default browser and says so; `devos app --browser` gives you a regular tab on purpose. The window remembers its size between launches.

### Documentation
- **README overhauled** into an accurate product homepage: value proposition, feature matrix (Understand/Learn/Build/Grow/System), quick start covering installer/portable-exe/pip/`devos app`, desktop-experience and update story, architecture diagram, AI philosophy, privacy & security, honest current-status table (v0.8.0, 356 tests, CI matrix), roadmap summary, contributing guide, and a screenshots placeholder (no fabricated images). Replaces text that still described the project as "early foundation (Phase 1)".

## [0.8.0] — 2026-06-11 ("DeveloperOS as a desktop product")
> Why this release exists: slices 13–15 complete the desktop story that v0.7.0's PWA
> foundation started. A non-technical user can now download a normal Windows installer from
> GitHub Releases, install DeveloperOS without admin rights, launch it from the Start Menu,
> and uninstall it cleanly (keeping their data) — no Python, no pip, no terminal. The D-0029
> desktop ladder is complete through step D; only the optional Tauri shell (step E) remains,
> and only if ever justified.

### Added
- **Project version bumped to 0.8.0**; shown in `devos --version` and the dashboard's System status.
- **A real Windows installer (slice 15).** DeveloperOS now installs like any other Windows program: run `DeveloperOS-Setup-<version>.exe`, click through a normal installer (no admin rights needed — it installs just for you), and get DeveloperOS in your **Start Menu** (desktop shortcut optional). Uninstall works from Windows Settings like any app — and it **never deletes your data**: your projects, tasks, notes and settings in `%APPDATA%\DeveloperOS` survive uninstall and reinstall. **Updates are manual and entirely in your control**: when a newer release appears on the GitHub Releases page, download the new Setup and run it — it upgrades in place. There is no auto-updater, no phoning home, no background anything. Developers build the installer with `packaging/build_installer.ps1` (Inno Setup, a build-time tool only). Heads-up: the binaries are unsigned (code-signing costs money), so Windows SmartScreen may ask for confirmation on first run. Also added: a proper `LICENSE` file (MIT) at the repository root.
- **`DeveloperOS.exe` — the packaging foundation (slice 14).** The repository can now build a **single-file Windows executable** (~10 MB) that needs no Python, no pip, and no terminal knowledge: double-click it and DeveloperOS starts (or reuses) the local dashboard and opens your browser — it wraps the `devos app` launcher, so all of its safety behaviors carry over. Developers build it with `packaging/build.ps1` (PyInstaller is a build-time tool only — DeveloperOS itself still has zero runtime dependencies). The exe carries the DeveloperOS icon (a new multi-size `devos.ico`, generated by the same dependency-free script as the app icons) and bundles the dashboard and database schema, so it works fully offline. Same code, same loopback-only security — just packaged. (Desktop ladder step C; the step-D installer will add a Start-Menu shortcut and an optional update check.)
- **`devos app` — launch DeveloperOS with one command (slice 13).** No more "start a server, then open a browser": `devos app` does the whole thing. It checks whether DeveloperOS is already running (and if so just opens it — you'll never end up with two copies), creates your data folder automatically on first run, starts the dashboard, waits until it's ready, and opens your browser at it. If the port is taken by another program it says so in plain language and suggests `--port`. Stop it any time with Ctrl+C. Loopback-only and offline as always; no new API surface. (Desktop ladder step B of D-0029 — and the entry point the future `DeveloperOS.exe` will wrap.)

## [0.7.0] — 2026-06-11 ("Installable DeveloperOS foundation")
> Why this release exists: slices 11+12 together turn DeveloperOS from "a dashboard you can
> reach in a browser" into "a product you can install and learn unaided" — onboarding earns a
> new user's trust in the first minute, and the PWA work gives DeveloperOS a real desktop
> identity (own window, Start-Menu icon). This is the foundation the D-0029 desktop ladder
> (launcher → exe → installer) builds on.

### Added
- **Install DeveloperOS like a real app (slice 12).** The dashboard is now an installable **Progressive Web App**: open it in Edge or Chrome and use the browser's "Install" button (in the address bar or menu) to get DeveloperOS as a **desktop app** — its own window, its own icon in the Start Menu and taskbar, no browser chrome. It comes with a proper app identity: a new DeveloperOS icon (a terminal-prompt mark in the dashboard's colors, generated by a dependency-free script in `tools/make_icons.py`) and a web app manifest. Everything stays exactly as local and private as before — installing adds no network access, no service worker, no background anything; the `devos serve` backend still runs on your machine only. (This is step A of the desktop roadmap recorded in D-0029: next come a one-command launcher and a single-file `devos.exe`.)
- **Project version bumped to 0.7.0**; shown in `devos --version` and the dashboard's System status.
- **First-run onboarding (slice 11): a welcome guide on Home.** The first time you open the dashboard (and any time your workspace is empty), the Home tab now starts with a friendly **"Welcome to DeveloperOS"** panel that explains, in plain language, what the app is — and states up front that **everything stays on your computer: no account, no API key, nothing is uploaded**. Below it, a **"Get started" checklist** walks you from zero to productive in six clicks: import your first project, search your code, ask a plain-English question, learn from a file, add a task or note, and see how AI is set up. Each step jumps straight to the right tab, and the checklist **ticks itself off as you actually do things** (imported projects, tasks and notes are detected automatically). Once you have data, you can hide the guide — and bring it back any time with a "Show the getting-started guide" link. Nothing else changed: no new endpoints, same security posture, fully offline.

## [0.6.1] — 2026-06-11 ("polished + accessible")
### Changed
- **Dashboard design system + accessibility pass (slice 10).** The dashboard now feels like one product instead of ten screens: a consistent spacing/typography/radius/motion token system, slightly larger base text (15px, nothing below 12px), comfortable button/input sizes (larger still on touch screens), and gentle 150ms hover/press feedback that is **fully disabled if your system asks for reduced motion**. ## [0.6.0] — 2026-06-11 ("feature-complete dashboard + first real AI")
### Added
- **Dashboard Meeting tab (slice 9) — the last CLI-parity gap is closed.** Paste meeting notes or a transcript into the new **Meeting** tab and get a **summary** (via the provider seam — offline mock by default), plus **action items detected deterministically from your own text** (bulleted lines under an "Action items"/"Next steps" heading, or lines starting with `TODO:` / `Action:` / `Next step:`). Review the detected items, untick any you don't want, and **create them as tasks in one click** — the bridge reuses the existing guarded `POST /api/tasks/create`, so no new write surface was added. The transcript is **never persisted** (same rule as the CV text). Powered by an inline `POST /api/meeting` with the same CSRF-token/origin/loopback/size guards as every other write; the action-item extractor lives in `modules/meeting.extract_action_items` and never calls a provider.
- **Ollama provider — the first real AI backend, local and free.** `devos/providers/ollama.py` implements the existing `AIProvider` seam against a **local Ollama daemon** (`http://127.0.0.1:11434` by default; configure with `DEVOS_OLLAMA_URL` / `DEVOS_OLLAMA_MODEL`). It is **stdlib-only** (`urllib`), needs **no API key**, costs **nothing**, and **nothing ever leaves your machine**. If the daemon isn't running it degrades gracefully — a clearly-labeled "[OLLAMA UNAVAILABLE]" result with plain-language fix instructions, never a crash. **Nothing changes by default:** the offline mock remains the default provider, the AI on/off toggle in Settings is untouched, and Ollama only runs if you deliberately select it (Settings tab now shows it as available instead of "coming soon").
- **Smarter retrieval (precision-first).** `ask`/`explain`/`debug`/`learn` grounding now tries an **all-terms (AND)** match first and widens to **any-term (OR)** only when nothing matches — so multi-word questions are grounded in tightly-matching code instead of chunks that share a single incidental word, while the old recall behavior is preserved as a fallback.
- **Secret-aware scanning.** `devos scan` (and dashboard import) now **skips credential-looking files before reading a single byte** — `.env*`, `*.pem`, `*.key`, `id_rsa*`, `.npmrc`, `.netrc`, `credentials*`, `secrets.*`, keystores and friends — so secrets can never reach SQLite or the search index. Scan results report a `skipped_secrets` count. (Closes SECURITY §2 "secret-aware indexing" PLANNED item.)
- **Continuous integration.** A free GitHub Actions workflow (`.github/workflows/ci.yml`) runs the full stdlib unittest suite on every push/PR across Python 3.11–3.13 on Linux and Windows.
- **Project version bumped to 0.6.0**; shown in `devos --version` and the dashboard's System status.

### Added earlier in this cycle, first shipped in 0.6.0 (no release was tagged before this one)
- **Dashboard Career tab (slice 8):** prepare for your job search without the CLI. A new **Career** tab has three plain-language sections: **Track a job application** (add companies/roles with a status — Saved → Applied → Interviewing → Offer — change status in one click, edit details, or remove an entry), **Interview prep** (pick a job and get likely interview questions grounded in the notes you saved for it — it tells you to add notes if there aren't any, instead of making things up), and **CV match check** (paste your CV and compare it against a job's notes or a pasted job description to see a **keyword-match score** plus which keywords you **cover** and which are **missing**). The CV check runs entirely on your machine and your CV text is **never stored**. Powered by read-only `GET /api/jobs` + `GET /api/jobs/interview` and guarded `POST /api/jobs/{create,update,delete}` + `POST /api/cv`, reusing the existing `devos job/cv/interview` engine.
- **Dashboard CRUD polish (slice 7):** the dashboard is now practical for everyday use. You can **delete** a task or a note (with a quick "are you sure?" confirm), and **assign a project** when adding a task or note (a simple dropdown). Tasks can be **renamed inline**. You can also **remove a whole project** from DeveloperOS via a clearly-marked **Danger zone** on the project page — it asks you to **type the project name** to confirm, and is explicit that it only removes the project from DeveloperOS (its index, plus that project's tasks and notes) and **never deletes the files on your disk**. All deletes go through the same CSRF-token/origin/loopback-guarded local API as other actions and are validated server-side.
- **Dashboard Learning Center (slice 6):** learn from your own code without the CLI. A new **Learn** tab lets you pick a **file or a topic** (optionally scoped to a project) and choose an explanation depth — **Beginner (plain English) / Intermediate / Advanced** — then **Explain it**, **Quiz me**, or **Give me exercises**, all grounded in your real indexed code with `file:line` sources. A **Check my understanding** box grades a free-text answer and gives feedback (strengths/weaknesses). When there isn't enough indexed code to be sure, it says so instead of guessing. Powered by read-only `GET /api/learn`, `GET /api/quiz`, `GET /api/exercise` and a guarded `POST /api/grade`, reusing the existing `devos learn/quiz/exercise/grade` engine; everything stays local and offline.
- **Dashboard Settings & AI Management (slice 5):** a new **Settings** tab lets you see and control how DeveloperOS runs — no config files needed. **System status** shows at a glance that you're **local-first** and **offline**, whether **AI is on**, which **provider** is active, the **version**, current **roadmap phase**, how many **projects are indexed**, and the **dashboard maturity**. **AI settings** let you turn AI features on/off and pick a provider — **Offline (Mock, default)**, **Ollama**, **Claude**, or **OpenAI** — each clearly badged **Local/Cloud** and **Free/May-cost-money**, with plain-language privacy and cost notes. Providers that aren't wired in yet are marked **Coming soon** and DeveloperOS keeps running the **offline mock**, so nothing leaves your machine until you deliberately enable a real provider in a future release. A prepared (disabled) provider-config panel previews where API key / endpoint / model will go. **Your API keys are never stored** by DeveloperOS — cloud providers read their key from an environment variable (e.g. `ANTHROPIC_API_KEY`), and the dashboard only ever shows whether a key is **detected** (never the value). Powered by read-only `GET /api/system` and `GET /api/settings` plus a guarded `POST /api/settings` (same CSRF-token/origin/loopback protection as other actions). Default remains mock + offline + free + no key.
- **Project version bumped to 0.5.0** (reflecting five shipped dashboard slices); shown in `devos --version` and in the dashboard's System status.

### Documentation
- Added **`docs/FUTURE_ROADMAP.md`** — a product-management view of where DeveloperOS is heading (v1.0/v2.0 vision; dashboard, AI, productivity, learning, career, and enterprise roadmaps; stretch goals; ideas backlog), with every idea prioritized Core / High Value / Nice-to-Have / Future Research. Planning only — nothing is implemented from it yet.

- **Dashboard Project Deep Dive / Study (slice 4):** open a project and understand it fast — built for learning and interview prep. From a project's detail view, **Study this project** opens a **Project Deep Dive** with friendly sections: **Start here** (the busiest files to open first), **Key files**, **How this works** (a plain-language overview of the project), **Questions to explore** (grounded study questions), and **Interview prep** (a practice checklist) — plus an **Ask about this project** box for plain-English questions. Powered by a read-only `GET /api/projects/study` that reuses the existing explain/quiz/search layers (no new analysis engine); everything stays local and offline.
- **Dashboard Debug Assistant tab (slice 3):** debug an error without the CLI. A new **Debug** tab gives you a large paste area — drop in an error message, stack trace, or chunk of logs, click **Analyze**, and get easy-to-read result cards: a summary (error + confidence), **what we think is going on** (plain-language root cause / fix, grounded in your own code), **where it points** (related files), and **sources** (file:line attribution). When there isn't enough indexed code to be sure, it says so instead of guessing. Powered by `POST /api/debug`, reusing the existing `devos debug` engine; read-only, the trace is never stored, and the request is protected by the same CSRF-token/origin/loopback guards as other actions.
- **Dashboard Projects tab (slice 2):** import and view projects without the CLI. A new **Projects** tab shows your projects as cards (file count, last scanned, folder), opens a **detail view** (name, folder, file count, last scanned, indexed status, what's inside), and offers a safe **Import a project** flow — type a folder path, read a plain-language explanation, then **confirm** before anything is read. Import scans **and** indexes the folder so it's immediately usable in Search & Ask, with a per-project **Re-scan** button. Powered by `POST /api/projects/scan` and `GET /api/projects/detail`, reusing the existing scan/index logic; the scan path is validated server-side and protected by the same CSRF-token/origin/loopback guards as other writes.
- **Action-oriented dashboard (slice 1):** the local web dashboard (`devos serve`) is now the primary, non-programmer-friendly way to work — not just a read-only view. Via lightweight tabs (Home · Tasks · Notes · Search & Ask) you can **create tasks, change a task's status / mark it done, add and edit notes, search your code, and ask plain-English questions** with cited sources — all in plain language with accessible labels. The CLI remains for power users.
- Local API gained guarded **write endpoints** (`/api/tasks/create|update`, `/api/notes/create|update`) plus read-only `/api/search`, `/api/ask`, `/api/explain`, all reusing the existing storage/modules layers. Writes are protected at the loopback boundary by a per-session **CSRF token** (`X-DevOS-Token`, delivered via `GET /api/session`), an **Origin allowlist**, a JSON content-type requirement, and a **64 KB** request cap; no CORS headers are emitted. Still 127.0.0.1-only, offline, mock AI provider.
- Project foundation: vision brief, phased roadmap (0–9), architecture, and source-of-truth state files.
- `devos` CLI with `init` (creates local data dir + SQLite DB, schema v1) and `status` (reports location, schema, provider, stored counts).
- Local-first SQLite storage layer with idempotent schema/migrations and an FTS5 search table reserved for indexing.
- AI provider abstraction with a built-in offline mock provider (no API key required).
- Editable install via `pip install -e .` exposing the `devos` command; 10 stdlib smoke tests.
- `devos scan <path>`: ingest a project folder — walks files with sensible ignore rules (a `.gitignore` subset, `node_modules`/venvs/build dirs, binary & oversized files), classifies each file into frontend/backend/db/api/auth/test/config/other, and records an idempotent file inventory (added/updated/unchanged/removed/skipped summary + per-type breakdown).
- `devos projects`: list registered projects with file counts and last-scanned time.
- `devos index <path>`: scan then build/refresh a searchable index — splits files into line-ranged chunks, mirrors them into a SQLite FTS5 index, and reindexes incrementally (only changed files, via content hash). Reports (re)indexed/unchanged/skipped files and chunk counts.
- `devos search <query>`: ranked keyword search (bm25) across indexed code/docs, returning `file:line-range` locations with highlighted snippets; supports `--project` and `--limit`.
- Storage upgraded to schema v2 (`files.indexed_hash`) with an upgrade-capable migration runner.
- `devos ask "<question>"`: grounded Q&A over your indexed project(s). Retrieves relevant chunks, answers via the configured AI provider (offline mock by default — no API key), and cites `file:line` sources. Clearly declines instead of guessing when there isn't enough indexed context.
- `devos explain [path]`: plain-language explanation of a specific file (from its indexed chunks) or a whole-project overview (file-type breakdown + notable files), with source citations.
- Added `docs/SECURITY.md` documenting the security-by-design architecture future phases must respect.
- `devos debug`: turn an error / stack trace / log (argument, `--file`, or piped stdin) into a grounded, structured diagnosis. Parses Python/Node/generic traces, locates the referenced files/lines **in your index**, retrieves related code, and asks the AI provider for Observed evidence / Likely root cause / Assumptions / Recommended fix / Verification steps — with `file:line` citations and an explicit confidence level. Declines instead of guessing when there's no evidence. Trace-named filesystem paths are never read (index-only lookups).
- `devos task` (add / list / show / set / rm): track tasks, bugs, and features with status (todo/in_progress/blocked/done), priority (low/medium/high), kind, milestone, and notes — optionally scoped to a project.
- `devos remember`: store long-term memory entries (decision / summary / preference / note) with tags; duplicate entries are de-duplicated.
- `devos recall`: search across your memory, tasks, and indexed code in one command (offline, with `file:line` for code). Storage upgraded to schema v3 (`tasks.priority`).
- `devos serve`: a local **dashboard** web UI (read-only, bound to 127.0.0.1) over the existing data — project overview, task status board, blocked items, recent activity, "where I left off", and a recall search box. Built on a stdlib `http.server` JSON API with a React (htm) front-end **vendored locally** (no build step, no npm, fully offline).
- `devos learn <path|topic> [--level eli5|intermediate|advanced]`: a learning assistant that explains your indexed code at the chosen depth, grounded in real source with `file:line` citations. Pass a file to explain that file, or a topic to retrieve relevant code. Declines instead of guessing when nothing relevant is indexed. Offline, mock provider by default.
- `devos quiz <path|topic> [--n N]`: generate N grounded review questions about a file or topic from your indexed code (default 5), with `file:line` sources. Declines instead of guessing when nothing relevant is indexed. Offline, mock provider by default.
- `devos meeting summarize <file>`: read a local transcript/notes file and produce a grounded summary with decisions and action items, citing the source file. Read-only, offline, mock provider by default; declines on empty input. CLI output is now UTF-8 console-safe (handles BOM/smart quotes/emoji without crashing on Windows).
- `devos plugins` + a plugin/extension system: external packages (entry-point group `devos.plugins`) and opt-in local files (`<data-dir>/plugins/*.py`, enabled with `DEVOS_ENABLE_LOCAL_PLUGINS=1`) can register new commands and AI providers. Plugin failures are isolated and listed by `devos plugins`. (Loading plugins runs third-party code — see SECURITY.md.)
- `devos job <add|list|show|set|rm>`: track job leads with company, role, url, status (saved/applied/interview/offer/rejected/closed), and notes.
- `devos cv <file> [--job ID]`: analyze a local CV/resume against a job's notes (or all tracked jobs) — offline keyword matching with matched/missing keywords and a coverage figure. No AI, fully local.
- `devos interview <job-id> [--n N]`: generate interview-prep questions grounded in a job lead's notes, citing the job as source. Declines when the job has no notes.
- `devos exercise <path|topic> [--n N]`: generate N grounded, hands-on practice exercises from your indexed code (default 3), with `file:line` sources.
- `devos grade <path|topic> --answer <text>|--answer-file <f> [--question <q>]`: evaluate your answer against the actual code — feedback with Strengths and Weaknesses, plus `file:line` sources. Read-only; declines when nothing relevant is indexed. Offline, mock provider by default.
- `devos docgen <type>`: generate grounded documentation — `readme`, `architecture`, `api`, `setup` (from indexed code + project facts) and `changelog`, `decisions`, `milestone` (from your memory/tasks). Prints to stdout by default; `--output PATH` writes a file (never overwrites without `--force`); appends a Sources footer (`file:line` for code, titles for records). Declines instead of guessing when there's nothing to ground on. Offline, mock provider by default.
